Object Identity

  • Each object is assigned an ID at its creation (think of a memory address).
  • This ID is unique and constant for this object as long as the object exists.
  • The built-in function id() can be used to retrieve the ID of an object.
1x = 1234
2y = x
3

4# x and y both point to the same object, 
5# therefore the IDs are the same.
6print(id(x) == id(y))  # True
7

8# x and y point to two different objects, 
9# therefore we expect x and y to have different IDs.
10x = 1234
11y = 5678
12print(id(x) == id(y))  # False
13

14# x and y point to two different objects (with the same value),
15# therefore we expect x and y to have different IDs.
16x = int("1234")  # integer 1234
17y = int("12" + "34")  # integer 1234
18

19print(id(x) == id(y))  # False

Identity operators

  • is and is not are comparison operators used to determine if the two operands have the same identity (i.e refer to the same object)
  • Note that this is NOT the same thing as checking for equality using == operator!
    • Using is operator means: are two variables referring to one and the same object?
    • Using == operator means: are two variables referring to objects that contain same data?
1x = int("1234")  # 1234
2y = int("12" + "34")  # 1234
3z = x 
4print(x == y)  # True
5print(x == z)  # True
1x = int("1234")  # 1234
2y = int("12" + "34")  # 1234
3z = x 
4print(x is y)  # False
5print(x is z)  # True
1x = 1234 
2y = 5678
3print(x is y)  # False
4print(x == y)  # False
5print(x is not y)  # True

Mutable vs Immutable objects

Immutable: the content of the object cannot be changed after the object has been created.

  • e.g. str, int, float, tuple

Mutable: the content of the object can be changed after its creation without changing its identity.

  • e.g. list, dict, set

Strings are immutable

Unlike lists, we cannot use the square brackets to modify a character in the string.
For example:

1s = "cats"
2s[0] = "r"  # TypeError: 'str' object does not support item assignment

All strings operations that seem to change a string actually create a new string.

1s = "cat"
2t = s
3print("Before:", s is t)
4s = s.replace('c','r')
5print("After:", s is t)
6print("s:", s)
7print("t:", t)
Output
Before: True
After: False
s: rat
t: cat

Lists are mutable

The following code does not create a copy of the list x.
It simply create a new variable name for the same list.

1x = [1, 2, 3]
2y = x  # new name y for same list
3

4print(x is y)  # True

Let us see some implications of this.

In the following illustrations, think about what is modified.

  • Whether a variable changes its value i.e. the variable refers to a different value
  • Whether a list object is modified i.e. some element of the list is changed.

What modifies a list?

  • Assigning a value to an element using its index. 
    1a = [1, 2, 3]
    2a[0] = 5
  • Using the slice assignment to modify the elements of a list. 
    1a = [1, 2, 3]
    2a[:2] = [4, 5]
  • Using methods like append(), insert(), remove() , pop(), sort(), etc.

What does not modify a list?

  • Slicing! It is a useful tool to create new lists out of an existing lists. 
    1a = [1, 2, 3]
    2b = a[:]  # makes a copy of a
    3print(a is b)  # False! The two variables refer to different objects.
  • The + and * operators create a new list 
    1a = [1, 2, 3]
    2b = a  # does not copy, just a new name for same list
    3b = b + [4]  # b now refers to a new list [1, 2, 3, 4]
    4             # a still refers to [1, 2, 3]
    5print(a is b)  # False

Packing vs Unpacking

When we create a string, a list, or a tuple, we are packing several elements into a single object.

1s = "cat"
2my_list = [5, 'a']
3my_tuple = (0, 3, 7)

Unpacking allows us to assign values in a string/list/tuple to multiple variables.
We must know the exact length of the string/list/tuple.

1s = "cat"
2a, b, c = s
3# a, b and c are all strings
4print(a)  # c
5print(b)  # a
6print(c)  # t
7

8my_list = [5, 'a']
9x, y = my_list
10print(x)  # 5
11print(y)  # a
12

13# Parentheses are optional in this context.
14my_tuple = 0, 3, 7
15x, y, z = my_tuple
16print(x)  # 0
17print(y)  # 3
18print(z)  # 7
19

20# Variables must match number of elements
21tup = 1, 2, 3 
22x, y = tup
23# ValueError: too many values to unpack (expected 2)

Multiple assignment using packing/unpacking on same line

1# We are creating a tuple on the right side and 
2# unpacking it into 3 variables.
3city, population, area = 'Montreal', 1704694, 431.5
4print(city)  # Montreal
5print(population)  # 1704694
6print(area)  # 431.5
