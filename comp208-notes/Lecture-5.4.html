5.4 — Tuples

Tuples

  • A tuple is an ordered collection of objects, like lists.
  • A tuple is immutable. A tuple object cannot change after it is created.
  • We create a tuple using parentheses () instead of square brackets [].
1tup = (1, 2, 3)
2print(type(tup))
3# <class 'tuple'>
4

5# tuple with only one item
6tup = (10,)  # comma is required!
7print(tup, type(tup))
8# (10,) <class 'tuple'>

We can use tuple function to convert other sequences such as lists and strings into a tuple.

1word = "apple"
2tup = tuple(word)
3print(tup)
4# ('a', 'p', 'p', 'l', 'e')
5

6primes = [2, 3, 5, 7]
7primes = tuple(primes)
8print(primes)
9# (2, 3, 5, 7)

Tuples are immutable

items cannot be added, removed or changed in a tuple.

Therefore, unlike lists, none of the operations that modify a tuple are allowed.

1x = (1, 1, 2, 3, 5, 8)
2x[3] = 100
3# TypeError: 'tuple' object does not support item assignment
4

5x[:2] = (11, 12)
6# TypeError: 'tuple' object does not support item assignment

In general, operations that do not modify a tuple are available.

1tup = (45, 23, 'abc') 
2

3# Indexing and slicing work the same way as lists.
4print(tup[1])
5# 23
6print(tup[1:])
7# (23, 'abc')
8

9print(len(tup))  # number of items in a tuple
10# 3
11

12# Similary min(), max(), and sum() functions work with tuples
13

14# Following methods are available for tuples
15print(tup.index("abc"))
16# 2
17print(tup.count(23))
18# 1

Since a tuple is a sequence, we can use it in a for loop just like a list:

1numbers = (1, 2, 3)
2

3for n in numbers:
4    print(n)
5

6# prints:
7# 1
8# 2
9# 3

Why use tuples?

If lists are more flexible than tuples, why should we use tuples?

  • Immutability is useful to avoid changing data by mistake.
  • We can use tuples as keys in a dictionary!
  • Programs are a bit faster when working with tuples.
