Sets

  • A set is a an unordered collection of immutable objects.
  • A set always contains unique elements, unlike lists and tuples which allow duplicates.
  • A set is unordered i.e. we cannot use indexing or slicing on a set object
1numbers = {1, 2, 3}
2print(numbers)  # {1, 2, 3}
3print(type(numbers))  # <class 'set'>
4

5# only unique values are kept
6numbers = {1, 2, 3, 1, 3}
7print(numbers)  # {1, 2, 3}
8print(len(numbers))  # 3

Other ways to create a set

1# a set can be created from any sequence
2# such as list, tuple or a string
3things = set([10, 42, "apple", 42])
4print(things)  # {'apple', 10, 42}
5

6word = "pineapple"
7letters = set(word)
8print(letters)  # {'p', 'n', 'l', 'i', 'e', 'a'}
9

10# create an empty set
11empty_set = set()
12print(len(empty_set))  # 0
13

14# This does not create an empty set!
15empty_dictionary = {}
16print(type(empty_dictionary))  # <class 'dict'>

Set elements must be immutable

A set can contain int, float, str, bool and tuple objects because they are all immutable.

But a set cannot contain a list because lists are mutable.

1# tuples are immutable so allowed in set
2points = {(1, 1), (3, 10), (3, 10)}
3print(points)  
4# {(3, 10), (1, 1)}
5

6# lists are mutable so not allowed
7points = {[1, 1], [3, 10], [3, 10]}  
8# TypeError: unhashable type: 'list'

Set operators and methods

1# set.add(x):
2#   Adds an element x to the set if x does not exist in the set. 
3#   Does not return anything.
4numbers = {1, 2, 3}
5numbers.add(20)
6print(numbers)  # {1, 2, 3, 20}
7

8numbers.add(3)
9print(numbers)  # {1, 2, 3, 20}
10

11

12# set.remove(x):
13#   Remove an element x from the set. Does not return anything.
14#   Throws KeyError if element x is not present in the set.
15numbers = {1, 2, 3}
16numbers.remove(2)
17print(numbers)  # {1, 3}
18numbers.remove(5)  # KeyError: 5
19

20# set.update(sequence):
21#   Adds all items from the sequence to the set.
22numbers = {1, 2, 3}
23numbers.update([10, 20, 2, 2, 3])
24print(numbers)  # {1, 2, 3, 10, 20}
25

26numbers.update(["a", "b"])
27print(numbers)  # {1, 2, 3, 10, 'a', 20, 'b'}
28

29

30# set.clear(): Remove all elements from this set.
31numbers = {1, 2, 3}
32numbers.clear()
33print(numbers)  # set()
34

35

36# in/not in operators to check if element exists in a set
37numbers = {1, 2, 3}
38print(2 in numbers)  # True
39print(5 not in numbers)  # True
40

41shapes = {'circle', 'square'}
42print("circle" in shapes)  # True

45# For the following methods, suppose A and B are sets.
46

47# A.intersection(B):
48#    Returns a new set that contains elements that are
49#    present in both A and B   
50odd = {3, 5, 7, 9, 25}
51squares = {4, 9, 25, 36}

61#    Returns a new set that contains elements that are
62#    present in  A or B or both

64x = {1, 2, 3}
65y = {2, 3, 5}
66all_numbers = x.union(y)

74# A.difference(B):
75#   Returns a new set that contains elements that are

Sets cannot be indexed or sliced because they are not ordered

1primes = {2, 3, 5, 7, 11}
2primes[4]  # TypeError: 'set' object is not subscriptable

But you can use for loop to iterate over the elements:

1# The order in which items will be printed is not defined
2# because sets are not ordered
3numbers = {10, 1, 5, 20}
4for n in numbers:
5    print(n)
Output
1
10
20
5

Why use sets?

  • Set are faster than lists and tuples, when inserting, removing and searching elements.
  • When ordering of elements is important or when elements are mutable, use lists or tuples
  • When only unique immutable elements need to be stored, use sets.
