Object Identity

  • Each object is assigned an ID at its creation (think of a memory address).
  • This ID is unique and constant for this object as long as the object exists.
  • The built-in function id() can be used to retrieve the ID of an object.
1>>> x = 500
2>>> y = x
3

4# x and y both point to the same object, 
5# therefore the IDs are the same.
6>>> id(x) == id(y)
7True
8

9# x and y point to two different objects, 
10# therefore we expect x and y to have different IDs.
11>>> x = 500
12>>> y = 400
13>>> id(x) == id(y)
14False
15

16# x and y point to two different objects (with the same value, but different!),
17# therefore we expect x and y to have different IDs.
18>>> x = 500
19>>> y = 500
20>>> id(x) == id(y)
21False

Identity operators

  • is and is not are comparison operators used to determine if the two operands have the same identity (i.e refer to the same object)
  • Note that this is NOT the same thing as checking for equality using == operator!
    • Using is operator means: are two variables referring to one and the same object?
    • Using == operator means: are two variables referring to objects that contain same data?
1>>> x = 500 
2>>> y = 500 
3>>> z = x 
4>>> x == y 
5True
6>>> x == z 
7True
1>>> x = 500 
2>>> y = 500 
3>>> z = x 
4>>> x is y 
5False
6>>> x is z 
7True
1>>> x = 500 
2>>> y = 300
3>>> x is y 
4False
5>>> x == y
6False
7>>> x is not y
8True

Mutable vs Immutable objects

Immutable: the content of the object cannot be changed after the object has been created.

  • e.g. str, int, float, tuple

Mutable: the content of the object can be changed after its creation without changing its identity.

  • e.g. list, dict, set

Strings are immutable

Unlike lists, we cannot use the square brackets to modify a character in the string.
For example:

1>>> s = "cats"
2>>> s[0] = "r"
3TypeError: 'str' object does not support item assignment

All strings operations that seem to change a string actually create a new string.

1s = "cat"
2t = s
3print("Before", id(s) == id(t))
4s = s.replace('c','r')
5print("After", id(s) == id(t))
6print("s:", s)
7print("t:", t)
Output
Before True
After False
s: rat
t: cat

Lists are mutable

The following code does not create a copy of the list x.
It simply create a new variable name for the same list.

1x = [1, 2, 3]
2y = x  # new name y for same list
3

4print(x is y)  # True

Let us see some implications of this.

What would the following examples print?

2y = x
3y = y + 3

1x = [5, 6, 7]
2y = x
3y[0] = y[0] + 3

2    x = x * 5

What modifies a list?

  • Assigning a value to an element using its index. 
    1a = [1, 2, 3]
    2a[0] = 5
  • Using the slice assignment to modify the elements of a list. 
    1a = [1, 2, 3]
    2a[:2] = [5]
  • Using methods like append(), insert(), remove() , pop(), sort(), etc.

What does not modify a list?

  • Slicing! It is a useful tool to create new lists out of an existing lists. 
    1a = [1, 2, 3]
    2b = a[:]  # makes a copy of a
    3print(a is b)  # False! The two variables refer to different objects.
  • The + and * operators create a new list 
    1a = [1, 2, 3]
    2b = a  # does not copy, just a new name for same list
    3b = b + [4]  # b now refers to a new list [1, 2, 3, 4]
    4             # a still refers to [1, 2, 3]
    5print(a is b)  # False

Tuples

  • A tuple is an ordered collection of objects, like lists.
  • A tuple is immutable. A tuple object cannot change after it is created.
  • We create a tuple using parentheses () instead of square brackets [].
1>>> tup = (1, 2, 3)
2>>> print(type(tup))
3<class 'tuple'>
4

5# tuple with only one element
6>>> tup = (10,)  # comma is required!
7>>> print(tup, type(tup))
8(10,) <class 'tuple'>

We can use tuple function to convert other sequences such as lists and strings into a tuple.

1>>> word = "apple"
2>>> tup = tuple(word)
3>>> print(tup)
4('a', 'p', 'p', 'l', 'e')
5

6>>> primes = [2, 3, 5, 7]
7>>> primes = tuple(primes)
8>>> print(primes)
9(2, 3, 5, 7)

Tuples are immutable

Elements cannot be added, removed or changed in a tuple.

Therefore, unlike lists, none of the operations that modify a tuple are allowed.

1>>> x = (1, 1, 2, 3, 5, 8)
2>>> x[3] = 100
3TypeError: 'tuple' object does not support item assignment
4

5>>> x[:2] = (11, 12)
6TypeError: 'tuple' object does not support item assignment

In general, operations that do not modify a tuple are available.

1>>> tup = (45, 23, 'abc') 
2

3# Indexing and slicing work the same way as lists.
4>>> print(tup[1])
523
6>>> print(tup[1:])
7(23, 'abc')
8

9>>> len(tup)  # number of elements in a tuple
103
11

12# Similary min(), max(), and sum() functions work with tuples
13

14# Following methods are available for tuples
15>>> tup.index("abc")
162
17>>> tup.count(23)
181

Since a tuple is a sequence, we can use it in a for loop just like a list:

1numbers = (1, 2, 3)
2

3for n in numbers:
4    print(n)
5

6# prints:
7# 1
8# 2
9# 3

Why use tuples?

If lists are more flexible than tuples, why should we use tuples?

  • Immutability is useful to avoid changing data by mistake.
  • We can use tuples as keys in a dictionary!
  • Programs are a bit faster when working with tuples.

Sets

  • A set is a an unordered collection of immutable objects.
  • A set always contains unique elements, unlike lists and tuples which allow duplicates.
  • A set is unordered i.e. we cannot use indexing or slicing on a set object
1>>> numbers = {1, 2, 3}
2>>> print(numbers)
3{1, 2, 3}
4>>> print(type(numbers))
5<class 'set'>
6

7# only unique values are kept
8>>> numbers = {1, 2, 3, 1, 3}
9>>> print(numbers)
10{1, 2, 3}
11>>> len(numbers)
123

7>>> word = "pineapple"
8>>> letters = set(word)
9>>> print(letters)

11

12# create an empty set
13>>> empty_set = set()
14>>> len(empty_set)

16

17# This does not create an empty set!
18>>> empty_dictionary = {}
19>>> print(type(empty_dictionary))

Set elements must be immutable

A set can contain int, float, str, bool and tuple objects because they are all immutable.

But a set cannot contain a list because lists are mutable.

1# tuples are immutable so allowed in set
2>>> points = {(1, 1), (3, 10), (3, 10)}
3>>> print(points)
4{(3, 10), (1, 1)}
5

6# lists are mutable so not allowed
7>>> points = {[1, 1], [3, 10], [3, 10]}
8TypeError: unhashable type: 'list'

Sets cannot be indexed or sliced because they are not ordered

1>>> primes = {2, 3, 5, 7, 11}
2>>> primes[4]
3TypeError: 'set' object is not subscriptable

But you can use for loop to iterate over the elements:

1# The order in which items will be printed is not defined
2# because sets are not ordered
3numbers = {10, 1, 5, 20}
4for n in numbers:
5    print(n)
Output
1
10
20
5

Why use sets?

  • Set are faster than lists and tuples, when inserting, removing and searching elements.
  • When ordering of elements is important or when elements are mutable, use lists or tuples
  • When only unique immutable elements need to be stored, use sets.

Dictionaries

Motivation

Suppose we would like to store the following enrollment data:

semesterno. of students
F2017816
W2018613
F2018709
W2019590
F2019744

We can do this using two lists for the two columns:

1semesters = ['F2017', 'W2018', 'F2018', 'W2019', 'F2019']
2students = [816, 613, 709, 590, 744]

What should we do if we want to add new data?

1semesters.append("F2020")
2students.append(550)
3# ['F2017', 'W2018', 'F2018', 'W2019', 'F2019', 'F2020']
4# [816, 613, 709, 590, 744, 550]

What if we want to modify the value for a specific semester?

1idx = semesters.index("W2018")
2students[idx] = 600

What we if try to add an entry for a semester that already exists?
List allows duplicates so it does not check if a semester already exists.

Use a dictionary!

  • You can think of an element of a dictionary as an ordered pair:
    • The first element of the pair is called a key.
    • The second element is referred to as the value.
  • A dictionary is called a mapping type because it maps key objects to value objects.
  • Elements in a dictionary are often called items.
1# A dictionary is created using a sequence of key-value pairs
2enrollment = {'F2017': 816, 'W2018': 613, 'F2018': 709,
3              'W2019': 590, 'F2019': 744}
4print(type(enrollment))
5# <class 'dict'>
6

7# Number of key-value pairs
8print(len(enrollment))
9# 5
10

11# This is an empty dictionary, not a set!
12empty_dict = {}
13print(len(empty_dict)) # 0

Note on keys and values

  • Keys
    • Have to be immutable objects.
    • Have to be unique in a dictionary. A dictionary cannot contain two items with the same key.
  • Values
    • Values can be of any type; both mutable and immutable values are allowed.
    • Many keys can map to the same value. i.e. values need not be unique.

Dictionary Lookup

With lists, we can access an element of the list through its index.

With dictionaries, we can access a value stored in the dictionary through the key associated with it.

1enrollment = {'F2017': 816, 'W2018': 613, 'F2018': 709,
2              'W2019': 590, 'F2019': 744}
3

4

5num_students = enrollment["F2018"]
6print(num_students)  # 709
7

8# Key must exist in the dictionary if we want to access its value
9print(enrollment["F2020"])  # KeyError: 'F2020'

We can add a new item by specifying a key and a value: dictionary[key] = value

18# {'F2018': 700, 'W2019': 590, 'F2019': 744, 'F2020': 800, 'W2020': 900}
